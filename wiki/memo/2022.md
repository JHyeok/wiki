이 말이 너무 인상깊어서 찾아보니 Egoless Programming를 10계명으로 요약한 내용의 2번에 이 내용이 있었다.

## 2022-02-12

처음에 Node.js로 사이드 프로젝트를 할 때는 저장소를 Mock 처리하지 않고 인메모리 데이터베이스를 이용해서 단위 테스트를 작성했다. 그리고 회사에서는 저장소를 Mock 처리를 해서 단위 테스트를 작성하고 있다.

오늘 단위 테스트에서 저장소를 Mock 처리해서 테스트하는 방식이 좋지 않다는 글을 읽었다. 기존에도 비슷한 글을 읽었지만 이번에 읽은 글은 너무 이유가 잘 설명되어 있어서 고개를 끄덕이게 되었다.

이제 단위 테스트를 작성할 때는 저장소를 Mock 처리하지 않고 직접 실제와 같은 DB를 사용한다거나 인메모리 DB를 사용해서 해결을 해야겠다.

## 2022-02-18

다른 프로젝트의 ESLint 의존성 때문에 WebStorm의 버전을 올려야 하는데 Git Flow 플러그인 때문에 올리지를 못하고 있다. 저번에 그냥 올렸다가 Git Flow 플러그인이 지원이 되지 않아서 터미널로 Git Flow를 사용했는데 불편했던 기억이 있다. 그래서 그냥 기다리려고 한다.

내가 사용하는 Git Flow 플러그인의 Fork 버전은 최신 버전을 지원하는데 Opher Vishnia님의 Git Flow 플러그인은 아직 지원을 하지 않아서 기다리고 있다. 이슈를 확인해보니 다른 사람들이 버전 문제를 해결한 PR들을 머지해주지 않고 있는데 이전의 이슈들에서 버전 지원은 본인이 하겠다는 댓글을 찾았다.

IDE의 버전이 올라가서 플러그인을 사용하지 못하는 것을 생각해보면 이러한 부분들은 VS Code가 더 편한 것 같다.
